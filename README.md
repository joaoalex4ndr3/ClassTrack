# Engenharia de Software — ClassTrack

## Sistema Inteligente de Acompanhamento Escolar

Esta branch contém os materiais referentes à disciplina de **Engenharia de Software** do projeto **ClassTrack**.

O objetivo desta branch é documentar a concepção do sistema, sua proposta, identidade visual, organização do desenvolvimento, casos de uso e apresentação geral da solução.

---

# Sobre o ClassTrack

O **ClassTrack** é uma plataforma web voltada para o acompanhamento acadêmico dentro do ambiente escolar.

A proposta do sistema é reunir em um só lugar informações importantes como:

* Notas;
* Frequência;
* Desempenho dos alunos;
* Indicadores acadêmicos;
* Controle de dados escolares.

O sistema busca auxiliar alunos, professores e gestores, tornando a gestão escolar mais organizada, eficiente e acessível.

---

# Quem Somos

O ClassTrack é uma iniciativa desenvolvida por estudantes do curso de **Análise e Desenvolvimento de Sistemas** do **SENAI Santa Rosália**.

A equipe tem como objetivo criar uma solução tecnológica voltada para o ambiente educacional, aplicando os conhecimentos adquiridos durante o curso em um projeto prático e funcional.

O foco principal é facilitar o acompanhamento acadêmico por meio de ferramentas digitais que auxiliem no controle de notas, frequência e desempenho dos alunos.

---

# Por que o nome ClassTrack?

O nome **ClassTrack** une duas ideias principais:

* **Class**: relacionado ao ambiente escolar, turmas e sala de aula;
* **Track**: relacionado ao acompanhamento, monitoramento e controle.

Assim, o nome transmite a ideia de acompanhar a vida acadêmica dos alunos de forma organizada, moderna e eficiente.

Além disso, é um nome simples, fácil de lembrar e com uma identidade forte para uma solução tecnológica educacional.

---

# Proposta do Projeto

O ClassTrack foi criado para resolver um problema comum em muitas instituições de ensino: a dificuldade em centralizar e acompanhar informações acadêmicas de forma rápida e organizada.

Muitas vezes, dados como frequência, notas e desempenho ficam separados ou são analisados tarde demais, dificultando ações preventivas por parte dos professores e da direção.

A proposta do ClassTrack é oferecer uma plataforma que ajude a escola a acompanhar essas informações de forma mais clara, facilitando a tomada de decisão.

---

# O que o ClassTrack oferece

A plataforma oferece recursos para facilitar o acompanhamento acadêmico dos alunos.

Entre os principais recursos estão:

* Controle de notas;
* Controle de frequência;
* Acompanhamento de desempenho;
* Visualização de indicadores;
* Organização das informações acadêmicas;
* Apoio aos professores e gestores;
* Melhor acompanhamento da vida escolar dos alunos.

---

# Ajuda e Controle

O ClassTrack foi pensado para melhorar o controle acadêmico e ajudar no acompanhamento dos estudantes.

Com uma plataforma organizada, professores podem identificar dificuldades mais cedo, alunos podem acompanhar melhor seu próprio desempenho e gestores podem visualizar informações importantes para a tomada de decisões.

A ideia é que o sistema funcione como uma ferramenta de apoio, ajudando a escola a agir antes que os problemas acadêmicos se agravem.

# Identidade Visual

A identidade visual do ClassTrack foi criada para transmitir tecnologia, educação, organização e confiança.

## Logo

A logo representa um sistema educacional digital.

Os principais elementos visuais são:

* **Notebook**: simboliza a tecnologia e a plataforma digital;
* **Chapéu de formatura**: representa a educação;
* **Gráficos e checklists**: representam o controle de notas, frequência e desempenho;
* **Tons de azul**: transmitem confiança, organização e modernidade.

## Cores

A identidade visual utiliza principalmente tons de azul, reforçando a proposta de um sistema moderno, confiável e organizado.

## Tipografia

A tipografia escolhida para a identidade visual foi pensada para manter uma aparência limpa, profissional e de fácil leitura.

# Design Inicial

O design inicial do ClassTrack foi planejado com foco em uma interface simples, moderna e intuitiva.

A proposta visual prioriza:

* Facilidade de navegação;
* Organização das informações;
* Visualização clara dos dados;
* Telas separadas por perfil de usuário;
* Layout responsivo;
* Experiência agradável para alunos, professores e direção.

O design serviu como base para o desenvolvimento das telas do sistema, incluindo login, painéis, dashboards e áreas de acompanhamento acadêmico.

# Casos de Uso

O sistema possui três atores principais:

* Aluno;
* Professor;
* Direção.

Cada ator possui funcionalidades específicas dentro da plataforma.

---

## Aluno

O aluno utiliza o sistema para acompanhar sua própria vida acadêmica.

### Principais ações do aluno

* Realizar login;
* Visualizar perfil;
* Consultar notas;
* Visualizar frequência;
* Visualizar ocorrências;
* Visualizar comunicados;
* Consultar horário de aulas;
* Receber alertas.

---

## Professor

O professor utiliza o sistema para gerenciar informações acadêmicas relacionadas às suas turmas.

### Principais ações do professor

* Realizar login;
* Gerenciar turmas;
* Registrar frequência;
* Lançar notas;
* Registrar ocorrências;
* Enviar comunicados;
* Visualizar relatórios.

---

## Direção

A direção utiliza o sistema para acompanhar dados gerais da instituição e apoiar a tomada de decisão.

### Principais ações da direção

* Realizar login;
* Cadastrar alunos;
* Editar alunos;
* Excluir alunos;
* Cadastrar professores;
* Editar professores;
* Excluir professores;
* Gerenciar turmas;
* Gerar relatórios;
* Consultar estatísticas;
* Emitir alertas;
* Gerenciar usuários;
* Configurar sistema.
